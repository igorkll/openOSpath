документация и список изменений к дополнению openOSfix для openOS

1.автозагрузка
1.1.теперь сласическая автозагрузка /autorun.lua или /.autorun.lua работает только из rootfs
для обеспечения максимальной безопастности,
но это может вылиться в проблемму при работе
на роботе так как софт робота монтируеться по средствам автозагрузки (/.autorun.lua)
и библиотека робота не будет доступна с самого начало это будет исправленно
в будуюших версиях патча а пока рекомендуеться написать свой скрипт монтировани
для робота, хотя если разработка безопастной автозагрузки затянеться он будет поставляться вместе с патчем
1.2./autoruns/user данная папка может содержать много автозагрузочьный скриптов
обратите внимания что выполняться они будут в произвольно порядке
1.3.приоритеты и порядок
приоритет работы (/autorun.lua и /.autorun.lua)
как и в ванильной openOS эти два файла не могут работать вместе приоритет стоит на файл "/autorun.lua"
сначала выполняться класический autorun скрипт
затем идут скрипты из папки /autoruns/system
и только потом скрипты из папки /autoruns/user
это сделано для того чтобы ваши программы из папки /autoruns/user
точьно работали верно

2.программы
2.1.player гуишная программа для проигрования midi через noise карту computronics
2.2.disk программа принимает на вход начала адреса файловай системы
затем переносит в первую точьку монтирования данной файловой системмы
2.3.oppm в комплект поставки патча уже входит пакетный менаджер oppm
2.3.eves программа выводит на экран все эвенты принимаетмые компьютерой
может использоваться при отладке или в иных целях
2.4.from простой устоновшик без лишних заморочек
копирует файлы с одного диска на другой и берет во внимания даже tmpfs
по умолчанию не копирует файлы начинаюшиеся с точьки но это легко исправить ключем -a
2.5.nchat программа для обшения позволяет обшаться по любому модему и соендененной карте
по тихнологии bigchat данная технология очень не безопастно и полностью онанимно
она использует ретрансляцию сообшения
так что настояший адрес отправителя очень трудно вычислить
и любой жилаюший может крашнуть почьти любую программу истользуюшию bigchat
отправив неправильный пакет хоть с другой точьки карты
главное чтобы боло достаточьно посредников
2.6.ntape гуишная программа для работы с tape_drive
имеет возможность перемотки и остоновли а также 2 кнопки с фиксацией
первая отвечает за авто перемотку ленты в начало 
вторая заставляет делать это для всех лент даже если не выбранна ниодна из них
2.7.tape стандартная программа tape постовлюеммая на дискете computronics идет в камплекте с патчем
2.8.ls была модифицирована чтобы показывать скрытые файлы

3.потоки
3.1.потоки закроються автоматически если она привязанны к радителю(вы не отвязывали их с помошью :detach())
то они закроються если
а.родитель будет закрыт с ошибкой
б.родитель закроетсья на os.exit
в.родитель завершиться сам или через return
3.2.закрытия потока,
поток не может быть закрык прирываниям вместо этого прерывания передет на программу переднего плана
поток будет закрыт если закроеться его радитель
или у обекта потока будет вызкан метод :kill()
если поток не имеет радителя а обект был утерен то закрить паток можно только перезегрузкой компьютера

4.таймеры и слушатили
4.1.если таймер или слушатель был создан программой переднего плана то он не будет завершен
при закрытии программы
4.2.если таймер или слушатель был создан потоком то он будет удален при смерти патока

5.отлючения прирываний
5.1.для отключения сочетания влавиш ctrl+alt+c необходимо задать event.superHook в false
5.2.для отключения сочетания влавиш ctrl+c необходимо задать event.hook в false

6.клавиатура
6.1.библиотака клавиотуры будет принимать сигналы только от первой клавиотуры системного экрана

7.экран и видео карта
7.1.системма будет работать и без этих элементов
7.2.при наличии конфига видеокарты системма проверяет наличия этой видеокарты если таковая имееться принивает настройки из конфига
в ином случаии ишет другую видеокарту если таковая нашлась то перезаписавыет конфиг на нее
7.3.если конфига нет а в компьютере есть хотябы одна видеокарта то конфиг будет записан за нее
7.4.в теже правила распостроняються и на мониторы
7.5.если при загрузке на мнгновения появлюеться ошибка слебует удолить конфиги видеокарты и экрана
7.6.в конфигах поддерживаються сокрашенные адреса
7.7.путь к конфигу видеокарты "/etc/gpu.lua" для повторного сохранения выховите saveGpuConfig()
7.8.путь к конфигу видеокарты "/etc/screen.lua" для повторного сохранения выховите saveScreenConfig()

8.библиотеки
8.1.bigchat 
8.1.1.библиотака с рестрансляцией сообшения между компьютеров
8.1.2.при получении сообшения генерирует евент big_chat c пераметрами переданными отправителям
8.1.3.метод send отправляет сообшения в содержать
8.1.4.благодоря алгоритму даюшему каждуму сообшению свой уникальный номер
ретрансляция не приводит в рекурсии и ddosу слушатили
8.1.5.bigchat обшяя сеть и для построения приватных сетей она не подходит
так как всеволиш один канал передачи данных между сетями обединит их
8.1.6.при написании программ с bigchat нужно юыть крайне осторожным
перед парсингом обязательно нужно проверять типы данных
иначе кто угодно сможет положить ваш сервес

8.2.env
8.2.1.библиотака содержит метод create
который создает и возврашает таблицу со всеми стандартными env
open computers за исключениям component и computer

8.3.midi2
8.3.1.явдяеться второй версией библиотеки midi, основанна на программе midi от сангара
8.3.2.метод create создает обект миди, первфй аргумен путь до midi файла, второй таблица инструментов
8.3.3.таблица инструментов может иметь всего один инструмент, тогда все каналы midi будут
идти в этот инструмент, а модет содержать несколько инструментов наприме для распределения нагрузке по развым устройства воспроизведения
8.3.4.таблица инструментов может содержать метод flush который вызовиться тосле того как все ноты данного такта попадут на свои инструменты
8.3.5.каждый инструмент это функция, первый аргумент это частота или нота(по умолчанию частота) второй задержка в секундах
8.3.6.третий аргумент нотный режим если он true то вместо частоты инструменты получают ноту
8.3.7.четвертый агрумент эфект pitch
8.3.8.пятый агрумент эфект speed, обратите внимания что он влияет только на скорость проигрования но не на длительность нот
8.3.9.шестой аргумент скорость нот чем выше тем ноты кароче и на оборот хотя и обозначен в обекте как notespeed
8.3.10.min и max может ограничевать верхний и нижний порог но работать будут только в месте есть нада использовать только один
задайте второй в безконечьтность, в случаи с min в минус безконечьтность
8.3.11.метой play начинает блокируюшия проигрования
8.3.12.метод loop начинает блокируюшия зациклинное проигрования
8.3.13.метод createThread создает поток который начинает проигрования и возврошает его
если первый аргумент истина то проигрования будет зациклинным
НЕ отвязывает поток от радителя при закрытии вашей программы музыка перестанет играть
если вам это не нужно поток необходиме отвязать с помошью :detach()

8.4.distfs позволяет предостовлять доступ к отдельным папка или вайловой системе в целов
8.4.1.работает по тихнологие bigchat
8.4.2.имеет два метода create и connect
8.4.3.1.create регистрирует публичьный доступ, аргументы:
индекс(строка с названия подключения нужна для подключения)
путь к папки с кторой будет производиться раздача
readonly если true пользователи не смогут произвести изменений в файловой системе
8.4.3.2.возврашает обект содержаший метод kill для прекшашения раздачи (НЕ явлюеться обектом потока) вызывать ныжно через . а не :
8.4.4.connect сначала индекс потом папка монтирования

8.5.vcomponent
8.5.1.не моя библиотака позволяет регистрировать виртуальные компоненты

9.bios данный bios разпологаеться на жеском диске и работает непосредственно от туда "/bios/bios.lua"
для устоновки его загрузчика необходимо ввести команду flash /home/superBiosInstallMe/loader.lua
